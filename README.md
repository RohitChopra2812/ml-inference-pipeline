# ML Inference Pipeline on Edge

A production-style ML inference pipeline built in C++ using ONNX Runtime, targeting edge deployment on Linux. Converts a PyTorch ResNet18 model to ONNX format and runs optimized inference with a C++ orchestration layer, achieving **71% latency reduction** via operator fusion and memory arena optimization.

---

## Project Structure

```
ml-inference-pipeline/
├── convert.py              # Export PyTorch ResNet18 → ONNX format
├── resnet18.onnx           # Exported ONNX model (generated by convert.py)
├── inference.cpp           # C++ inference engine using ONNX Runtime
├── CMakeLists.txt          # CMake build system
├── test_inference.py       # Python pytest regression test suite
└── imagenet_classes.txt    # 1000 ImageNet class labels
```

## Setup

### Prerequisites
- Python 3.10+
- CMake 3.14+
- GCC / Clang (C++17)
- ONNX Runtime C++ SDK

### 1. Create Python virtual environment

```bash
python3 -m venv ml_inference_env
source ml_inference_env/bin/activate
pip install torch torchvision onnxruntime onnxscript pytest numpy
```

### 2. Export model to ONNX

```bash
python convert.py
```

This downloads pretrained ResNet18 weights and exports `resnet18.onnx`.

### 3. Download ONNX Runtime C++ SDK

```bash
curl -L https://github.com/microsoft/onnxruntime/releases/download/v1.18.0/onnxruntime-osx-arm64-1.18.0.tgz -o onnxruntime.tgz
tar -xzf onnxruntime.tgz
```

> For Linux x86_64, replace `osx-arm64` with `linux-x64`.

### 4. Build C++ inference engine

```bash
mkdir build && cd build
cmake ..
make
```

### 5. Run inference

```bash
./build/inference
```

**Expected output:**
```
Model loaded successfully
Loaded 1000 class labels
Inference ran successfully

Top 5 Predictions (optimized session):
1. [623] letter opener  (score: 4.68703)
2. [499] cleaver  (score: 3.96145)
3. [111] nematode  (score: 3.87513)
4. [813] spatula  (score: 3.79958)
5. [596] hatchet  (score: 3.79037)
```

## Performance Results

| Mode | Avg Latency | Optimization |
|---|---|---|
| Baseline | 38.67 ms | No optimization |
| Optimized | 10.91 ms | Operator fusion + memory arena + multithreading |
| **Improvement** | **71.7%** | |

### Optimizations applied:
- **Operator fusion** (`ORT_ENABLE_ALL`) — fuses Conv+BatchNorm+ReLU chains into single kernels
- **CPU Memory Arena** — reuses memory allocations, eliminates malloc/free overhead per run
- **Multithreading** — parallelizes ops across 4 CPU cores (`SetIntraOpNumThreads(4)`)

## Key Implementation Details

### C++ Inference Engine (`inference.cpp`)
- Uses `Ort::Session` to load and run ONNX models
- Input tensor shape: `[1, 3, 224, 224]` (batch, channels, height, width)
- Output: 1000-class probability scores
- Post-processing: sorts scores to extract top-5 predictions with class names

### CMake Build (`CMakeLists.txt`)
- C++17 standard
- Links against `libonnxruntime.dylib`
- Single `add_executable` target for clean builds

### Model Export (`convert.py`)
- Downloads pretrained ResNet18 (ImageNet weights)
- Exports with `opset_version=12` for broad compatibility
- Input/output nodes named `"input"` / `"output"` for C++ API alignment
